## 二、欄位規格與驗證規則

| 欄位 | 類型 | 必填 | 驗證規則 | 錯誤訊息 |
|------|------|------|---------|---------|
| 公司名稱 | text | ✓ | 不可空白，最多 50 字 | 請填寫公司名稱 |
| 統一編號 | text | ✓ | 8 位數字，台灣統編驗證演算法 | 請填寫有效的統一編號（8 位數字） |
| 您的姓名 | text | ✓ | 不可空白，最多 20 字 | 請填寫您的姓名 |
| 詢問事項 | textarea | 選填 | 最多 500 字 | — |

### 統一編號驗證邏輯

台灣統編採用加權驗證演算法，前端實作 JavaScript 驗證：

```javascript
// 台灣統一編號驗證（標準演算法）
function validateTaxId(id) {
  if (!/^\d{8}$/.test(id)) return false;
  const weights = [1, 2, 1, 2, 1, 2, 4, 1];
  let sum = 0;
  for (let i = 0; i < 8; i++) {
    let n = parseInt(id[i]) * weights[i];
    sum += Math.floor(n / 10) + (n % 10);
  }
  return sum % 10 === 0 || (id[6] === '7' && (sum + 1) % 10 === 0);
}
```

**驗證時機：** `onBlur`（離開欄位時）+ 送出前再驗證一次，不要在打字時即時顯示錯誤（避免打擾使用者）

## 三、送出後的使用者體驗

### 流程

```
老闆填完表單
    │
    ▼
點擊「送出預約申請」
    │
    ▼
前端驗證（統編格式、必填欄位）
    │
    ├── 驗證失敗 → 在對應欄位下方顯示紅色錯誤訊息，捲動到第一個錯誤欄位
    │
    └── 驗證通過 → 按鈕變灰 + 顯示「送出中...」（防止重複送出）
                    │
                    ▼
                API 呼叫（Formspree / EmailJS）
                    │
                    ├── 成功 → 表單區域替換為感謝畫面
                    │
                    └── 失敗 → 顯示錯誤訊息「送出失敗，請稍後再試或直接聯繫我們」
                                按鈕恢復可點擊狀態
```

## 五、後台技術方案評估

### 方案比較

| 方案 | 費用 | 設定難度 | 自動回覆 | 統編驗證 | 適合度 |
|------|------|---------|---------|---------|------|
| **Formspree** | 免費（50次/月）/ $10/月無限 | 低（HTML form 直接整合） | Pro 版支援 | 前端 JS | ★★★★☆ |
| **EmailJS** | 免費（200次/月）/ $15/月無限 | 中（需 JS SDK） | 支援 | 前端 JS | ★★★★☆ |
| **Web3Forms** | 免費（250次/月） | 低（類似 Formspree） | 支援 | 前端 JS | ★★★☆☆ |
| 自建後端 | 主機費用 | 高 | 完全自訂 | 前後端 | ★★☆☆☆ |

### 建議方案：**Formspree**（免費版起步）

**理由：**
1. 靜態網站直接整合，不需要後端程式
2. HTML form 加一個 `action` 屬性即可運作
3. 流量初期不會超過免費額度（50次/月 = 每月 50 個詢問）
4. 若日後量增，升級 Pro（$10/月）即可開啟自動回覆
5. 後台可看到所有表單送出紀錄，方便追蹤

**實作方式：**
```html
<form action="https://formspree.io/f/{YOUR_FORM_ID}" method="POST">
  <input type="text" name="company" required>
  <input type="text" name="tax_id" required>
  <input type="text" name="name" required>
  <textarea name="message"></textarea>
  <button type="submit">送出預約申請</button>
</form>
```

**統編驗證：** 透過前端 JavaScript 在送出前執行，Formspree 只負責轉寄 Email。

## 六、手機版特別考量

目標受眾（傳統企業老闆）有相當比例使用手機瀏覽：

- 表單欄位全部 `width: 100%`，避免需要橫向捲動
- 統一編號欄位設 `inputmode="numeric"`，手機會自動跳出數字鍵盤
- 送出按鈕全寬（`width: 100%`），拇指易點擊
- 錯誤訊息字體不小於 14px
