# 冷杉 Cedar — 網站頁面架構

**Issue #6 | 撰寫日期：2026-03-25**
**依據：競爭對手分析報告（Issue #5）+ 品牌規範（CLAUDE.md）**

---

## 設計原則

根據競爭對手分析的核心發現，本架構遵循三個原則：

1. **每一頁只做一件事** — 訪客在任何頁面都清楚自己在這裡能得到什麼
2. **動線終點永遠是「預約諮詢」** — 所有頁面的 CTA 最終指向同一個行動
3. **老闆的問題先於冷杉的服務** — 先讓他感到「這說的是我」，再說「我們能做什麼」

---

## 頁面清單

| 頁面 | 路徑 | 核心目的 |
|------|------|---------|
| 首頁 | `/` | 讓老闆三秒內感到「這是給我的」，引導預約 |
| 關於我們 | `/about` | 建立信任：我們是誰、為什麼做這件事 |
| 我們的解法 | `/solution` | 說清楚「六個後台」是什麼、怎麼運作 |
| 捷行案例 | `/case` | 用真實數字消除疑慮，讓老闆算得出帳 |
| 預約諮詢 | `/contact` | 唯一轉換頁，最低門檻完成預約 |

**共 5 個頁面**（不含系統頁）

---

## 各頁面核心目的詳述

---

### 首頁 `/index.html`

**核心目的：** 在 3 秒內讓「面臨接班、想退場」的傳統企業老闆感到「這說的是我的問題」，然後引導他採取下一步。

**訪客心理狀態（進站前）：**
- 可能從搜尋「家族企業接班」「中小企業人事精簡」等關鍵字進來
- 對 AI 半信半疑，但對「公司靠我一個人撐著」的焦慮是真實的
- 第一個問題：「這間公司是幫我的嗎？」

**頁面要完成的任務：**
1. 用老闆聽得懂的語言說出他的問題
2. 給一個直覺可信的數字（成本對比）
3. 引導他看案例或直接預約

**轉換動線：**
首頁 → 預約諮詢（主要）／捷行案例（次要）

---

### 關於我們 `/about.html`

**核心目的：** 回答「你們是誰？我能信任你嗎？」

**訪客心理狀態（進站前）：**
- 已對首頁感興趣，想確認這間公司是真的
- 台灣傳統企業老闆對「新創」有顧慮，需要人的故事不是公司資料

**頁面要完成的任務：**
1. 說清楚冷杉是誰、為什麼做這件事（創辦人視角）
2. 讓老闆感到「這些人懂我」，而不是「這些人要賣我東西」
3. 傳遞服務的核心承諾（接管，不是建議）

**轉換動線：**
關於我們 → 預約諮詢 / 我們的解法

---

### 我們的解法 `/solution.html`

**核心目的：** 說清楚「六個後台到底是什麼」，讓老闆看完就算得出帳，並消除「AI 我聽不懂」的心理障礙。

**訪客心理狀態（進站前）：**
- 想知道「你們具體做什麼」
- 擔心「這很複雜，我不懂技術」
- 在評估「這值不值這個價錢」

**頁面要完成的任務：**
1. 用具體場景說出六個後台職能（採購、人事、財務、客服、倉儲、行政）
2. 說明導入方式（不需要老闆懂技術）
3. 說清楚費用結構（月費 + 建置費）
4. 呼應「兩個月回本」的邏輯

**轉換動線：**
我們的解法 → 捷行案例（我想看真實效果）／預約諮詢（我想了解更多）

---

### 捷行案例 `/case.html`

**核心目的：** 用一個真實的台中家居老闆故事，讓看頁面的老闆算出帳、消除最後的疑慮。

**訪客心理狀態（進站前）：**
- 有興趣但還沒決定相信
- 「有沒有真的做過？效果怎樣？」
- 需要一個「跟我很像的人做過、有用」的依據

**頁面要完成的任務：**
1. 說捷行的故事（導入前的問題、導入過程、導入後的改變）
2. 用數字說話：70–80% 成本壓縮、月費 11 萬 vs 市場 35–51 萬、兩個月回本
3. 讓老闆感到「這個人跟我一樣，他做到了」

**轉換動線：**
捷行案例 → 預約諮詢（最強的轉換頁，訪客在這裡最容易決定行動）

---

### 預約諮詢 `/contact.html`

**核心目的：** 讓老闆用最低心理門檻完成預約，並確保冷杉收到完整的詢問資訊。

**訪客心理狀態（進站前）：**
- 已有足夠信任，準備採取行動
- 但對「被推銷」仍有警戒心
- 希望「預約」是輕鬆的，不是簽約

**頁面要完成的任務：**
1. 說清楚「這 15 分鐘諮詢是什麼」（免費、不推銷、只是了解）
2. 提供簡單表單（公司名稱、統一編號、姓名、詢問事項）
3. 送出後讓老闆感到「我做了一件對的事」（感謝畫面）
4. 觸發 Email 通知

**轉換動線：**
預約諮詢 → 感謝畫面（送出後）

## 頁面之間的動線圖

```
[外部流量]
    │
    ▼
[首頁]
    ├──→ [捷行案例] ──→ [預約諮詢] ──→ [感謝畫面]
    ├──→ [我們的解法] ──→ [捷行案例] ──→ [預約諮詢]
    ├──→ [關於我們] ──→ [預約諮詢]
    └──→ [預約諮詢] ──→ [感謝畫面]
```

**主要轉換路徑（預期最多訪客走這條）：**
首頁 → 捷行案例 → 預約諮詢

**次要路徑（深度了解型訪客）：**
首頁 → 我們的解法 → 捷行案例 → 預約諮詢

## 共用元件

所有頁面共用以下元件：

### 導覽列（Navigation）
- Logo（左）
- 頁面連結：關於我們 / 我們的解法 / 捷行案例
- CTA 按鈕：「立即預約諮詢」（右，金色，每頁都有）

### 頁尾（Footer）
- 公司名稱與簡短說明
- 快速連結
- 聯絡方式
- 版權聲明

### CTA 區塊（可重用 Section）
- 標題：「準備好讓公司不再只靠你嗎？」
- 副標：「15 分鐘免費諮詢，沒有義務」
- 按鈕：「立即預約諮詢」
- 此區塊出現在每一個頁面的底部（預約諮詢頁除外）

## 技術備註

- **靜態網站**：純 HTML / CSS / JS，無需後端框架
- **表單處理**：Formspree 或 EmailJS（Issue #8 詳細評估）
- **Email 通知**：送出後自動寄送給冷杉，包含公司名稱、統編、姓名、詢問事項
- **響應式設計**：手機、平板、桌機均需支援（傳統企業老闆可能用手機瀏覽）
