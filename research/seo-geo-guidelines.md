## 二、H1/H2/H3 標題關鍵字使用原則

### H1
- 全頁唯一，包含最重要的主關鍵字或情境句
- 用老闆的語言，不用術語（❌「AI 驅動後台自動化」→ ✅「讓公司繼續轉，即使你退了」）
- 字數：10–25 字

### H2
- 每頁 3–6 個，包含次要關鍵字
- **洞見文章**的 H2 優先使用問句格式，提高 GEO 被引用機率：
  - ✅「台灣中小企業後台外包費用多少？」
  - ✅「老闆退休後公司能繼續跑嗎？」
  - ❌「費用說明」（太平，無關鍵字）

### H3
- 細項說明，無需硬塞關鍵字，但避免重複 H2 的用詞

## 三、關鍵字密度建議

| 類型 | 建議密度 | 說明 |
|------|---------|------|
| 主關鍵字 | 1–2% | 每 100 字出現 1–2 次即可，過高會被判為 keyword stuffing |
| 次要關鍵字 | 0.5–1% | 自然融入即可 |
| 長尾關鍵字 | 文章開頭 + 結尾各一次 | 集中在最重要的位置 |

**原則：寫給人看，不是寫給機器看。** 如果讀起來像在塞關鍵字，就代表塞太多了。

## 四、GEO 高引用價值段落寫作格式

### 規則一：每篇文章必須有一段「直接回答型」段落

格式：
```
[關鍵問題]，答案是[數字/結論]。
以[具體案例]為例，[結果描述]，[費用或時間]。
```

範例：
> 「台灣中小企業聘用完整後台團隊，市場行情每月約 35–51 萬元。導入 AI 後台服務後，以捷行國際家居為例，月費降至 11 萬，後台成本壓縮 70–80%，建置費 50 萬，兩個月回本。」

### 規則二：每篇文章開頭加 TL;DR 摘要（100 字以內）

```markdown
> **本文重點：** [一句話說明文章主旨，含數字]。
> 適合閱讀對象：[描述目標讀者]。
```

### 規則三：FAQ 段落格式

每篇洞見文章末尾加 2–3 個 FAQ（用 H3 + 回答），同步加入 FAQPage Schema：

```markdown
### 常見問題

#### [問題一，使用目標關鍵字]
[直接回答，2–4 句，含具體數字]

#### [問題二]
[回答]
```

## 六、內部連結策略

### 原則
- 每頁至少 2–3 個內部連結
- 連結文字（Anchor Text）要包含目標頁的關鍵字，不用「點這裡」或「了解更多」

### 建議連結地圖

| 來源頁 | 連結到 | Anchor Text 建議 |
|--------|--------|----------------|
| 首頁 | 捷行案例 | 「看捷行國際家居的真實成效」 |
| 首頁 | 我們的解法 | 「了解六個後台職能怎麼運作」 |
| 我們的解法 | 捷行案例 | 「看費用壓縮 70% 的真實案例」 |
| 我們的解法 | 預約諮詢 | 「預約 15 分鐘評估你的公司」 |
| 捷行案例 | 預約諮詢 | 「你的公司也可以這樣，預約諮詢」 |
| 洞見文章 | 我們的解法 | 「了解冷杉怎麼接管後台」 |
| 洞見文章 | 捷行案例 | 「看捷行國際家居的導入成效」 |
| 洞見文章 → 洞見文章 | 相關主題文章 | 含關鍵字的自然語句 |

## 七、洞見文章 SEO Checklist（每篇發布前對照）

```
□ H1 包含主關鍵字（文章標題）
□ Meta Title ≤ 60 字元，含主關鍵字
□ Meta Description ≤ 155 字元，含主關鍵字 + 數字或行動句
□ 文章開頭有 TL;DR 摘要（≤ 100 字）
□ H2 標題至少 3 個，其中 1 個以上用問句格式
□ 文章中段有一段「直接回答型」段落，含具體數字
□ 文末有 FAQ 段落（2–3 個問答）
□ 加入 FAQPage Schema
□ 所有圖片有 Alt Text
□ 至少 2 個內部連結（Anchor Text 含關鍵字）
□ 文末有 CTA 區塊（連結至預約諮詢）
□ 字數建議：800–1,500 字（針對長尾關鍵字）
```
