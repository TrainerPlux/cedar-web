## 三、GEO 定期審查清單

每次新增文章或更新頁面前，對照以下清單：

### A. 新文章發布清單
```
□ H1 包含主關鍵字（字元數 ≤ 30 字）
□ Meta Title ≤ 60 字元，含主關鍵字
□ Meta Description ≤ 155 字元，含主關鍵字 + 數字
□ 文章開頭有 TL;DR 摘要（≤ 100 字，含具體數字）
□ 文中有一段「直接回答型」段落
   格式：「[問題]，答案是[數字]。以[案例]為例，[結果]。」
□ H2 標題至少 3 個
□ 文末有 FAQ 段落（2–3 則問答）
□ FAQPage Schema 已加入
□ 至少 2 個內部連結（Anchor Text 含關鍵字）
□ 文末有 CTA 連結到 contact.html
□ 加入 Article Schema（含 datePublished、dateModified）
□ 加入 sitemap.xml 條目並更新 lastmod
```

### B. 每月 GEO 效果追蹤
```
□ 測試：在 ChatGPT / Perplexity / Gemini 搜尋以下問題，記錄冷杉是否出現：
  - 「台灣有哪些公司幫傳統企業接班？」
  - 「中小企業後台外包費用多少？」
  - 「台灣 AI 後台服務有哪些真實案例？」
  - 「家族企業老闆退休公司怎麼辦？」
□ 記錄結果，與上月對比
□ 若某問題連續 2 個月未出現，強化對應頁面的直接回答段落
```

### C. 頁面更新清單
```
□ 更新 sitemap.xml 的 <lastmod> 日期
□ 如有數字或費用變動，同步更新 Schema Markup
□ 確認所有內部連結仍有效
```
